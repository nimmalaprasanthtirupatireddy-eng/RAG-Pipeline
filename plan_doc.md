**What is RAG?**

RAG(Retrieval Augmented Generation) is a technique of connecting external data to Large Language Models to imporve the accuracy in the responses.

**How does it work?**

1. **Data Collection**

Data is collected from various pdf files consist of text, tables and images. Using llamaParse package we collected the data from pdf files. 
And we do summarize for the table data and image data. Added to that in the end of text file. and it saved as text in text file.

2. **Text Splitting To Chunks**

Text from text file is access and split in to chunks using RecursiveCharacterTextSplitter. And **Chunk_size = 800 and chunk_overlap = 150**

3. **Embedding and Vector Store**

Here we used HuggingFaceEmbeddings to convert text chunks into embeddings. In that the model is **sentence-transformers/all-MiniLM-L6-v2**.
And Used **FAISS** to store the embeddings. 

    **Why FAISS?**

    FAISS is a library for efficient similarity search and clustering of dense vectors. It has speed and efficiency in searching the similar vecotrs.

    **why sentence-transformers/all-MiniLM-L6-v2?**

    It is a lightweigth model and it is efficient in semantic similarity, it has 384 dimensions. it has trained on Large Dataset of sentences and it is good at collecting the context of the text.

    Actually In the transformer architecture, there is this thing self-attention mechanism. It is used to calculate the attention weights of the input sequence. And predict the next word. 

4. **LLM Chain**

Here we used Qwen3:1.7b model to generate the response. It is a open source LLM model. It is trained on Large Dataset of text. It is good for text generation and summarization.

Here we used Langchain Ollama to connect the qwen3:1.7b model to our RAG pipeline.

Here we used Langchain PromptTemplate to create a prompt for the LLM model. And we used Langchain StrOutputParser to parse the response from the LLM model. And we used Langchain RunnablePassthrough to pass the response from the LLM model to the RAG chain.

5. **Retrieve Content**

Here we used Langchain as_retriever to retrieve the content from the vector store. It uses cosine similarity to find the most similar chunks to the question. And it returns the top_k chunks. In our case top_k = 5.

Actually we can use top_p also if the question is not relevant to the content.

6. **Response Generation**

The answer is generated by the LLM based on the retrieved content and the question.
